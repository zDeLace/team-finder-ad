# Стартовый набор и задание

Длительность: 10 минут

У будущего веб-приложения уже есть готовый фронтенд — это HTML-шаблоны и код JavaScript. Файлы хранятся в папках `static` и `templates_var1/2/3`. 
Репозиторий вы сможете склонировать себе на компьютер после того, как прочтёте уроки этой темы.

Содержание подпапок `static`:

- В папке `js` — JavaScript-код, который можно использовать для фронтенда.
- В папке `css` — готовые стили, которые можно использовать для проекта.
- В папке `fonts` лежит шрифт, который рекомендуется использовать в проекте.
- В папке `images` подготовлена картинка, которую рекомендуется использовать для аватара по умолчанию.

Содержание подпапок `templates`:

- `includes` — частички шаблонов, которые можно использовать в разных других шаблонах.
  - `svg` — картинки в формате svg.
- `projects` — шаблоны, относящееся к отображению проектов.
- `users` — шаблоны, относящееся к отображению страниц пользователей.

Так же в корне лежат файлы `docker-compose.yml` с настройками оркестрации Докера и `.env_example` с примером переменных окружения.
В файле `requirements.txt` содержится пример зависимостей, которые могут быть в проекте.

## Задание

Ваша задача как бэкенд-разработчика — написать бэкенд с помощью Django для веб-приложения «Team Finder» и проверить, что проект работает в соответствии с требованиями.

Проект должен быть полностью рабочим и соответствовать описанию из следующих уроков.

Можно менять или удалять любые файлы, которые уже есть в репозитории.

У вас нет жёстких рамок по структуре и содержанию кода, однако есть технические условия общего плана, которые должны быть соблюдены.

## Каким требованиям должна соответствовать инфраструктура проекта

- Проект должен использовать базу данных PostgreSQL.
- В Django-проекте должен быть файл `requirements.txt` со всеми зависимостями.

## Что ещё должно быть в проекте

Протестируйте веб-приложение: нужно создать несколько пользователей и добавить хотя бы по одному проекту от имени каждого пользователя.
Пример того, как может выглядеть готовый проект, можно посмотреть тут: http://team-finder.tech

Также желательно добавить и автоматические тесты.
