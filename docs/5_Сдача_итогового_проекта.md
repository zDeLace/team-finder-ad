# Сдача итогового проекта

Длительность: 1800 минут

В отличие от предыдущих проектов, этот будут проверять ревьюеры – специальные сотрудники Яндекс Практикума, которые знают, как должен выглядеть хорошо выполненный проект.

Рекомендуем перед отправкой проекта на ревью пройтись по двум чек-листам. Это поможет убедиться, что все пункты задания выполнены.

## Общий чек-лист

- Проект «Team Finder» полностью функционален и работает без ошибок.
- Проект использует базу данных PostgreSQL.
- В корне проекта есть файл `requirements.txt` со всеми зависимостями.
- (не обязательно) Настроен CI через GitHub Actions для автоматической проверки кода.
- В репозитории есть несколько тестовых пользователей, и у каждого создан как минимум один проект.
- База данных или проект целиком разворачивается и работает с использованием Docker Compose.
- Данные сохраняются в `volumes`, чтобы не теряться при перезапуске контейнеров.
- Код соответствует стандарту PEP 8 (длину строки можно увеличить до 100 символов)
- В файле `README.md` есть инструкция для ревьюера по запуску проекта.

## Чек-лист по функциональности

Проверьте, что все страницы и функции работают в соответствии с техническим заданием.
С примером готового проекта можно ознакомиться здесь: http://team-finder.tech

### Общая функциональность (для всех вариантов)

**Страницы и навигация:**

- [ ] Главная страница: отображает список проектов.
- [ ] Страница регистрации: доступна форма с полями (имя, фамилия, email, пароль).
- [ ] Страница входа: доступна форма для аутентификации (email, пароль).
- [ ] Страница пользователя: отображает публичный профиль.
- [ ] Страница редактирования профиля: доступна владельцу профиля.
- [ ] Страница проекта: отображает детальную информацию о проекте.
- [ ] Страница создания/редактирования проекта: доступна для создания и изменения проекта.
- [ ] Страница со списком пользователей: отображает всех зарегистрированных пользователей.
- [ ] Страница смены пароля: доступна авторизованным пользователям.

**Главная страница (список проектов):**

- [ ] Проекты отображаются в виде карточек.
- [ ] Карточка содержит: автора (ФИ и аватар), название, краткое описание, дату публикации.
- [ ] Клик по автору или названию перенаправляет на соответствующие страницы (пользователя или проекта).
- [ ] Проекты отсортированы по дате (новые сверху).
- [ ] Реализована постраничная пагинация (12 проектов на странице).
- [ ] Для авторизованных пользователей: видна иконка «Добавить в избранное» и кнопка «Создать проект».

**Регистрация и авторизация:**

- [ ] Пользователь может зарегистрироваться, указав обязательные поля.
- [ ] После регистрации происходит перенаправление на страницу входа.
- [ ] Пользователь может войти в систему по email и паролю.
- [ ] Реализован выход из системы.

**Страница пользователя:**

- [ ] Отображается информация: ФИ, аватар, описание, контакты (email, телефон, GitHub).
- [ ] Отображается список проектов, созданных пользователем.
- [ ] Владелец профиля видит кнопки «Редактировать профиль», «Добавить проект».
- [ ] Работает функция «Поделиться профилем».

**Редактирование профиля:**

- [ ] Владелец может изменять свои личные данные (ФИ, аватар, описание, контакты).
- [ ] Изменения сохраняются после нажатия кнопки «Сохранить».

**Страница проекта:**

- [ ] Отображается детальная информация: название, автор, дата, описание, статус.
- [ ] Авторизованный пользователь может присоединиться к проекту.
- [ ] Владелец проекта видит кнопки «Редактировать проект», «Завершить проект».
- [ ] У владельца отображается список участников проекта.
- [ ] Работает функция «Поделиться ссылкой» на проект.

**Создание и редактирование проекта:**

- [ ] Форма содержит поля: название (обязательное), описание, статус.
- [ ] При редактировании поля предзаполнены текущими данными.
- [ ] Кнопка «Опубликовать»/«Сохранить изменения» сохраняет проект.
- [ ] После сохранения происходит перенаправление на страницу проекта.

**Страница со списком пользователей:**

- [ ] Пользователи отображаются в виде карточек (ФИ, аватар).
- [ ] Реализована кнопка «Посмотреть контакты».
- [ ] Пользователи отсортированы по дате регистрации (новые сверху).
- [ ] Реализована постраничная пагинация (12 профилей на странице).

**Права доступа:**

- [ ] **Гость (аноним):** может просматривать проекты, профили пользователей, но не может создавать контент или откликаться.
- [ ] **Авторизованный пользователь:** может создавать и управлять своими проектами, откликаться на чужие, редактировать свой профиль.
- [ ] **Администратор:** имеет полные права на управление пользователями и проектами через админ-панель.

### Функциональность по вариантам

#### Вариант 1: Избранное и фильтрация пользователей

- [ ] Авторизованный пользователь может добавлять/удалять проекты в избранное с главной страницы и страницы проекта.
- [ ] Существует страница «Избранное», доступная только владельцу и отображающая его избранные проекты.
- [ ] На странице списка пользователей (`/users/list/`) работает фильтрация по критериям: «Авторы избранных проектов», «Авторы проектов, в которых я участвую», «Пользователи, которым нравятся мои проекты», «Участники моих проектов».

#### Вариант 2: Навыки пользователей и фильтрация

- [ ] На странице пользователя отображается блок «Навыки».
- [ ] Владелец профиля может добавлять и удалять навыки без перезагрузки страницы.
- [ ] При добавлении навыка работает автодополнение; можно создать новый навык, если его нет в списке.
- [ ] На странице списка пользователей (`/users/list/`) есть панель для фильтрации по навыкам.
- [ ] Применение фильтра (`?skill=<Название>`) оставляет в списке только пользователей с выбранным навыком.
- [ ] Активный фильтр подсвечивается, есть кнопка для сброса фильтра.

#### Вариант 3: Навыки проектов и фильтрация

- [ ] На странице проекта отображается блок «Необходимые навыки».
- [ ] Владелец проекта может добавлять и удалять навыки для своего проекта без перезагрузки страницы.
- [ ] При добавлении навыка работает автодополнение; можно создать новый навык.
- [ ] На странице списка проектов (`/project/list/`) есть панель для фильтрации по необходимым навыкам.
- [ ] Применение фильтра (`?skill=<Название>`) оставляет в списке только проекты с выбранным навыком.
- [ ] Активный фильтр подсвечивается, есть кнопка для сброса фильтра.
